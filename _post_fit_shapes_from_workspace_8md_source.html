1 Post-fit Distributions {#post-fit-shapes-ws}
2 =========================================
3 
4 **File**: CombineTools/test/PostFitShapesFromWorkspace.cpp
5 
6 The program `PostFitShapesFromWorkspace` is used to dump a set of pre- and post-fit distributions from a datacard. There is no need to specify which categories or processes to extract, this will be done automatically and so should work for any shape-based analysis. The effect of all normalisation and shape uncertainties is evaluated. In addition, there is an option to calculate post-fit/pre-fit scale factors for each background and the total background uncertainty in each category. This numbers can then be used as input for producing post-fit control distributions.
7 
8 
9 The following options are available:
10 
11  Configuration:
12  -w [ --workspace ] arg The input workspace-containing file [REQUIRED]
13  --dataset arg (=data_obs) The input dataset name
14  -d [ --datacard ] arg The input datacard, only used for rebinning
15  -o [ --output ] arg Name of the output root file to create [REQUIRED]
16  -f [ --fitresult ] arg Path to a RooFitResult, only needed for postfit
17  -m [ --mass ] arg Signal mass point of the input datacard
18  --postfit [=arg(=1)] (=0) Create post-fit histograms in addition to pre-fit
19  --sampling [=arg(=1)] (=0) Use the cov. matrix sampling method for the post-fit
20  uncertainty
21  --samples arg (=500) Number of samples to make in each evaluate call
22  --print [=arg(=1)] (=0) Print tables of background shifts and relative
23  uncertainties
24  --freeze arg Format PARAM1,PARAM2=X,PARAM3=Y where the values X and Y
25  are optional
26  --covariance [=arg(=1)] (=0) Save the covariance and correlation matrices of the
27  process yields
28  --skip-prefit [=arg(=1)] (=0) Skip the pre-fit evaluation
29  --skip-proc-errs [=arg(=1)] (=0) Skip evaluation of errors on individual processes
30  --total-shapes [=arg(=1)] (=0) Save signal- and background shapes added for all
31  channels/categories
32  --reverse-bins arg List of bins to reverse the order for
33 
34  Example usage:
35  PostFitShapesFromWorkspace -d htt_mt_125.txt -w htt_mt_125.root -o htt_mt_125_shapes.root \
36  -m 125 -f mlfit.root:fit_s --postfit --sampling --print
37 
38 
39 \warning The option `--sampling` is not used by default, but you should probably use it if you are producing post-fit distributions. Without this the uncertainty will be calculated assuming no correlations between nuisance parameters - this would be extremely rare in an typical fit.
