1 Python Interface {#python-interface}
2 ====================================
3 
4 [TOC]
5 The C++ CombineHarvester class and a number of other tools have a python interface available. The module containing this interface can be imported like any other CMSSW python module:
6 
7  import CombineHarvester.CombineTools.ch as ch
8 
9 The sections below summarise the methods that are currently supported.
10

12 ==========================================

18

20 
21  cb = ch.CombineHarvester()
22  cb_shallow_copy = cb.cp()

87 Filtering {#py-filtering}
88 =========================
89 All of the basic filter methods can be called and chained in a similar way in both interfaces. For example:
90 
91 C++:
92 
93  cb.bin({"muTau_vbf"}).era({"7TeV", "8TeV"}).signals().process({"ggH"}, false);
94 
95 Python:
96 
97  cb.bin(['muTau_vbf']).era(['7TeV', '8TeV']).signals().process(['ggH'], False)
98 
99 The generic filter methods (FilterAll, FilterObs, FilterProcs, FilterSysts) have also been adapted to accept python functions, e.g.
100 
101 C++:
102 
103  cb.FilterAll([](ch::Object const* obj) {
104  return obj->mass() == "110" || obj->mass() == "145";
105  });
106 
107 Python:
108 
109  cb.FilterAll(lambda obj : obj.mass() in ['110', '145'])
110 
111 Set producers {#py-sets}
112 ========================
113 
114 All basic set producer methods are available.
115 
116 C++:
117 
118  set<string> bins = cb.bin_set();
119  for (auto p : cb.process_set()) {
120  ...
121  }
122 
123 Python:
124 
125  bins = cb.bin_set()
126  for p in cb.process_set():
127  ...
128 
129 The generic methods are available too, and accept a generic function object.
130 
131 C++:
132 
133  set<string> bins = cb.SetFromAll(std::mem_fn(&ch::Object::bin));
134  set<string> some_set = cb.SetFromProcs([](ch::Process const* p) {
135  return p->process() + "_" + p->bin();
136  });
137 
138 Python:
139 
140  bins = cb.SetFromAll(ch.Object.bin)
141  some_set = cb.SetFromProcs(lambda x : x.process() + '_' + x.bin())

143 Modifications {#py-modifications}
144 =================================
145 
146 The `GetParameter`, and `UpdateParameters` methods that use `ch::Parameter` objects are not available (and may be deprecated soon anyway). The `UpdateParameters` method taking a `RooFitResult` is available however.
147 
148 C++:
149 
150  TFile f("mlfit.root");
151  RooFitResult *res = (RooFitResult*)f.Get("fit_s");
152  cb.UpdateParameters(*res);
153 
154 Python:
155 
156  import ROOT as R
157  f = R.TFile("mlfit.root")
158  res = f.Get("fit_s")
159  cb.UpdateParameters(res)
160 
161 The generic ForEach methods are also available and can take arbitrary python functions as input.
162 
163 C++:
164 
165  void SwitchToSignal(ch::Process const* p) {
166  if (p->process() == "ggH_hww125") p->set_signal(true);
167  }
168  cb.ForEachProc(SwitchToSignal);
169 
170 Python:
171 
172  def SwitchToSignal(p):
173  if p.process() == 'ggH_hww125': p.set_signal(True)
174  cb.ForEachProc(SwitchToSignal)

202 Datacard creation {#py-creation}
203 ================================
204 
205 Creating observation and process entries is supported.
206 
207 C++:
208 
209  cb.AddObservations({"*"}, {"htt"}, {"8TeV"}, {"mt", "et"}, {{0, "0jet"}, {1, "1jet"}})
210  cb.AddProcesses({"*"}, {"htt"}, {"8TeV"}, {"mt", "et"}, {"ZTT", "ZL", "ZJ"}, {{0, "0jet"}, {1, "1jet"}}, false)
211 Python:
212 
213  cb.AddObservations(['*'], ['htt'], ['8TeV'], ['mt', 'et'], [(0, "0jet"), (1, "1jet")])
214  cb.AddProcesses(['*'], ['htt'], ['8TeV'], ['mt', 'et'], ['ZTT', 'ZL', 'ZJ'], [(0, "0jet"), (1, "1jet")], False)
215  ## or with default values:
216  cb.AddProcesses(procs = ['ZTT', 'ZL', ZJ'], bin = [(0, "0jet"), (1, "1jet")], signal=False)
217 
218 As is bin-by-bin creating and merging, but note that these methods have been deprecated in favour of the standalone ch::BinByBinFactory class (see below).
219 
220 C++:
221 
222  cb.MergeBinErrors(0.1, 0.5);
223  cb.AddBinByBin(0.1, true, cb);
224 
225 Python:
226 
227  cb.MergeBinErrors(0.1, 0.5)
228  cb.AddBinByBin(0.1, True, cb)
229 
230 The creation of Systematic entries with the `AddSyst` method is supported, though has a slightly different syntax due to the heavy template usage in the C++ version.
231 
232 C++:
233 
234  cb.cp().process({"WH", "ZH"}).AddSyst(
235  cb, "QCDscale_VH", "lnN", SystMap<channel, era, bin_id>::init
236  ({"mt"}, {"7TeV", "8TeV"}, {1, 2}, 1.010)
237  ({"mt"}, {"7TeV", "8TeV"}, {3, 4, 5, 6, 7}, 1.040)
238  ({"et"}, {"7TeV"}, {1, 2, 3, 5, 6, 7}, 1.040)
239  ({"et"}, {"8TeV"}, {1, 2}, 1.010)
240  ({"et"}, {"8TeV"}, {3, 5, 6, 7}, 1.040));
241 
242 Python:
243 
244  cb.cp().process(['WH', 'ZH']).AddSyst(
245  cb, "QCDscale_VH", "lnN", ch.SystMap('channel', 'era', 'bin_id')
246  (['mt'], ['7TeV', '8TeV'], [1, 2], 1.010)
247  (['mt'], ['7TeV', '8TeV'], [3, 4, 5, 6, 7], 1.040)
248  (['et'], ['7TeV'], [1, 2, 3, 5, 6, 7], 1.040)
249  (['et'], ['8TeV'], [1, 2], 1.010)
250  (['et'], ['8TeV'], [3, 5, 6, 7], 1.040))
251 
252 
253 The ExtractPdfs, ExtractData and AddWorkspace methods are not currently supported.
